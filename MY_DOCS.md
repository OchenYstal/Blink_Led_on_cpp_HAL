# [Blink_Led_on_cpp]

**Data:** 09.04.26

## Общее описание
Программа управления состояниях пинов(светодиодов) с использованием кнопки и внешним прерыванием.

## Классы и функции

### Class Led 
**Назначения:** Создания обьектов для управления светодиодами на пинах.
**Зачем нужен:**  Что бы не инициализировать каждый пин отдельно для светодиода.

**Метод:** void on()
**Что делает:**  Подает питание на пин(HIGH)
**Зачем/Почему:** Обертка на HAL_GPIO_WritePin(SET)

**Метод:** void off()
**Что делает:** Подает питание на пин(LOW)
**Зачем/Почему:** Обертка на HAL_GPIO_WritePin(RESET)

**Метод:** void Init()
**Что делает:** Инициализация каждого Led'а 
**Зачем/Почему:** Обертка над большим и не компактным кодом

### Class Button
**Назначения:** Создания обьектов для управления кнопками(внешними) на пинах 
**Зачем нужен:** Что бы не инициализировать каждый пин для Led'а отдельно и сделать код компактным

**Метод:** void init()
**Что делает:** Инициализация каждой кнопки
**Зачем/Почему:** Что бы не инициализировать каждый пин для кнопки отдельно и сделать код компактным

**Метод:** void onPress()
**Что делает:** функция выполняет логику реакции на события(кнопки) (можно использовать для других обьектов 
                Например: Таймер -> сработал период -> onPeriodElapsed()
                          UART -> Принял байт -> onByteReceived() 
                          )
**Зачем нужна:** Для упращения написания логики для кнопки(с учетом прерывания EXTI) + нужна для работы globalDispatch

**Метод:** bool matches(uint16_t triggerPin) const
**Что делает:** Сравнивает пин который вызвал прерывание(triggerPin), с пином который отвечает данный обьект(_pin)
**Зачем нужна** В массиве buttonList лежат разные кнопки на разных пинах. Он проверяет какой пин вызвал прерывание для того что бы вызвать(дать) ей функцию onPress()

**Метод:** static void globalDispatch(uint16_t pin)
**Что делает:** Получает от железа номер сработавшего пина, обходит весь список созданных кнопок, найдя нужный пин, вызывает у него onPress()
**Зачем нужна** Это мост для связи между C функции, C++ обьектов. Так как C функции не знают про классы и обьекты, поэтому еще + ставится static

**Метод:** void registerButton(Button* btn)
**Что делает:** Добавляет указатели на обьект кнопки в статический массив buttonList
**Зачем нужна:** Нужен для того что бы globalDispatch вообще знал какие кнопки есть.
